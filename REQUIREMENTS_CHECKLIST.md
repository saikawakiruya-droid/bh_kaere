# A-Maze-ing 要件確認チェックリスト（v2.2）

## III. 共通ルール（General Rules）

- [ ] Python 3.10 以降で実装している
- [ ] flake8 のコーディング規約に準拠している
- [ ] 例外を try-except で適切にハンドリングし、クラッシュしない（未処理例外で落ちると「動作不良」扱い）
- [ ] ファイル/接続などのリソースは context manager 等で確実に解放している（リーク防止）
- [ ] 関数の引数・戻り値・変数に型ヒントを付けている（typing モジュール使用）
- [ ] mypy による静的型チェックをエラーなしで通過する
- [ ] 関数・クラスに PEP 257 準拠の docstring（Google または NumPy スタイル）を付けている

## III.2 Makefile

- [ ] `Makefile` をプロジェクトに含めている
- [ ] `install` : pip / uv / pipx 等で依存関係をインストールする
- [ ] `run` : メインスクリプトを実行する
- [ ] `debug` : pdb などのデバッガでメインスクリプトを実行する
- [ ] `clean` : 一時ファイル/キャッシュ（`__pycache__`, `.mypy_cache`）を削除する
- [ ] `lint` : `flake8 .` と `mypy . --warn-return-any --warn-unused-ignores --ignore-missing-imports --disallow-untyped-defs --check-untyped-defs` を実行する
- [ ] `lint-strict`（任意）: `flake8 .` と `mypy . --strict` を実行する

## III.3 追加ガイドライン

- [ ] テストプログラムを作成している（pytest/unittest、エッジケースを網羅。提出・採点対象外）
- [ ] Python 成果物を除外する `.gitignore` を含めている
- [ ] （推奨）venv / conda で依存関係を分離している

## IV.2 実行方法（Usage）

- [ ] `python3 a_maze_ing.py config.txt` で実行できる
- [ ] メインファイル名は必ず `a_maze_ing.py`
- [ ] 引数は設定ファイル1つのみ（ファイル名は変更可）
- [ ] すべてのエラーを丁寧に処理する（不正な設定・ファイル未検出・構文エラー・実現不可能なパラメータ等）
- [ ] 予期せぬクラッシュをせず、ユーザーに明確なエラーメッセージを出す

## IV.3 設定ファイル形式（Configuration file）

- [ ] 1行あたり `KEY=VALUE` 形式
- [ ] `#` で始まる行はコメントとして無視する
- [ ] 必須キー `WIDTH`（迷路の幅・セル数）に対応
- [ ] 必須キー `HEIGHT`（迷路の高さ）に対応
- [ ] 必須キー `ENTRY`（入口座標 x,y）に対応
- [ ] 必須キー `EXIT`（出口座標 x,y）に対応
- [ ] 必須キー `OUTPUT_FILE`（出力ファイル名）に対応
- [ ] 必須キー `PERFECT`（完全迷路かどうか）に対応
- [ ] （任意）seed / algorithm / display mode などの追加キーに対応
- [ ] デフォルト設定ファイルを Git リポジトリに用意している

## IV.4 迷路の要件（Maze Requirements）

- [ ] 迷路はランダム生成される
- [ ] seed による再現性を実装している（必須）
- [ ] 各セルは 0〜4 個の壁を持つ（北・東・南・西の各方位）
- [ ] 入口と出口が存在し、互いに異なり、迷路の範囲内にある
- [ ] 全体が連結し、孤立セルがない（「42」パターンを除く）
- [ ] 外周（外側の境界）に壁がある
- [ ] 隣接セル間の壁の整合性が取れている（片側だけ壁があるのは禁止）
- [ ] 大きな開放領域がない：通路の幅は最大2セル（2x3, 3x2 は可、3x3 は不可）
- [ ] 視覚表示時に、完全に閉じた複数セルで描かれた「42」が見える
- [ ] 迷路が小さすぎて「42」を描けない場合、コンソールにエラーメッセージを表示する

### PERFECT=True の場合
- [ ] 入口〜出口に**唯一の経路**のみ存在する（ループなしの完全迷路）

### PERFECT=False（デフォルト）の場合 — Pac-Man 風ボード
- [ ] 全通路が到達可能（完全連結、ボード全体がパックガムで埋められ攻略可能）
- [ ] 四隅と中央が開いた通路になっている（隅=ゴースト/スーパーパックガム、中央=プレイヤー開始）
- [ ] 独立した経路（ループ）が**最低2つ**ある（完全迷路や壁1枚除去=単一ループは不可）
- [ ] 行き止まりは稀に留める（数個は許容。行き止まりゼロは理想でボーナス対象）

## IV.5 出力ファイル形式（Output File Format）

- [ ] 1セルにつき16進数1桁で壁を表現する
- [ ] ビット割当：bit0(LSB)=北, bit1=東, bit2=南, bit3=西
- [ ] 壁が閉=1、開=0
- [ ] セルを行ごとに格納（1行=1行分のセル）
- [ ] 空行の後、3行に分けて以下を出力：入口座標／出口座標／入口から出口への最短有効経路（N,E,S,W の文字列）
- [ ] すべての行が `\n` で終わる
- [ ] `maze_analyzer.py`（提供スクリプト）で壁エンコードの整合性・perfect/playable 判定を確認できる
