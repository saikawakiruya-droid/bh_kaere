# 敵対的レビュー記録（examples/adversarial/）

迷路ジェネレーターを「壊す・仕様の穴を突く」つもりで意地悪な入力を **1件ずつ実際に実行**し、
攻撃意図と観測結果を記録したもの。各ケースは `adv_NN_<slug>.txt`（使った設定）と
`adv_NN_<slug>_result.txt`（攻撃意図・観測・判定・プログラム出力）の対で構成される。

- **HELD** … 優雅に処理（クラッシュせず、正しく生成 or 正しく拒否）
- **OMITTED** … サイン「42」が配置できず省略（非致命的、仕様IV.4どおり）
- **DEFECT** … 実際の不具合（クラッシュ／壁不整合／検証すり抜け）

## 発見された懸念（不具合ではないが記録すべき点）

### 所見1: 1マス幅 + PERFECT=False は「プレイアブル不成立」を回避できない（adv_03, 02）
1列（または1行）幅ではループが原理的に作れないため、PERFECT=False を要求しても
プレイアブル条件（独立ループ2本以上・四隅中央の通路化）を満たせない。
- 我々の verifier は正しく「条件未達」を報告する。
- 課題付属 maze_analyzer は同じ盤面を「PERFECT maze（単一経路）」と判定する（矛盾ではなく、
  1マス幅の連結迷路は定義上ちょうど単一経路のため）。
- 仕様上 WIDTH/HEIGHT は「正の値（1以上）」が有効で、最小サイズ規定やこのケースの禁止は無い。
- 対応方針は保留（設定時に弾くと仕様外の制約追加になるため）。関連: 2列幅なら HEIGHT>=3 で成立する
  （2xN の独立ループ数 = N-1）。
- 追加の設計上の穴（adv_03）: `a_maze_ing.run` は生成後検証(`validate`)の失敗を warning として
  表示するが exit code には反映せず **exit 0（成功）** を返す。幾何的にプレイアブル不可能な盤面が
  「成功」として出力され、構造のみ検査の `verification.cli` や `maze_analyzer` も素通りする。
  検証失敗を非ゼロ終了に反映するか、少なくとも「playable 要求だが未達」を明確に区別することを検討。

### 所見2: サインが置けないときの省略が「無言」の経路がある（adv_09, #27関連）
SIGN=空 など一部の経路ではサインが省略されても明示メッセージが出ない
（サイズ不適合・配置不能は warning を出すが、空サインは静かに reserved 空になる）。
観測性の観点で、省略理由を一貫して記録できると望ましい。braiding 側の corridor 開口失敗は
別途 #27 で stderr 警告を追加済み。

## ファズ試験(ランダム500件)との整合
別途実施したランダム500件（各設定値をランダム化 → maze_analyzer 検証）でも、
クラッシュ0・壁不整合0・解析不能0 で本記録と整合。その過程で、正常盤面において
`count_dead_ends` が「サインに囲まれて生じた行き止まり」を除外せず verifier が過剰に
厳格になる不一致を発見し、Issue #35 として登録済み。
