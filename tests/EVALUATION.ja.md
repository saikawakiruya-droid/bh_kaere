# プロジェクト A-Maze-ing の評価スケール

`このチームでは 2 名の学生を評価します`

Git リポジトリ: `git@vogsphere-v2.42to...`

## はじめに

- 評価プロセス全体を通じて、礼儀正しく、敬意を持ち、建設的な態度を保ってください。
  コミュニティの健全さはそれにかかっています。

- 成果物に不具合があれば、評価される本人（またはグループ）と一緒に特定してください。
  見つけた問題については時間を取って議論し、話し合ってください。

- 課題の指示や機能の範囲について、相手の理解が自分と異なる可能性があることを
  念頭に置いてください。常に広い心を持ち、できる限り誠実に採点してください。
  ピア評価が真剣に行われて初めて、この教育手法は意味を持ちます。

## ガイドライン

- 採点対象は、学生またはグループの Git リポジトリにある成果物のみです。

- その Git リポジトリが本当にその学生またはグループのものか、必ず確認してください。
  対象のプロジェクトの成果物であることを確かめ、`git clone` は空のフォルダで
  実行してください。

- 公式リポジトリの内容以外のものを評価させるための、悪意あるエイリアスが
  仕込まれていないか注意深く確認してください。

- 予期せぬ事態を避けるため、採点を補助するスクリプトを使う場合は、評価する側と
  評価される側の双方がその内容を確認済みであることを確かめてください。

- 評価する側の学生がまだこのプロジェクトを終えていない場合、口頭試問を始める前に
  課題文（subject）全体を読むことが必須です。

- 空のリポジトリ、動作しないプログラム、Norm 違反、不正行為などを報告するには、
  このスケールにあるフラグを使ってください。これらの場合、採点はそこで終了し、
  最終評価は 0 点（不正行為の場合は -42 点）になります。ただし不正行為を除き、
  （たとえ完成していなくても）成果物について議論を続け、失敗の原因となった問題を
  特定し、今後同じ過ちを繰り返さないようにすることを推奨します。

- 口頭試問の間、プログラムが予期せず・早期に・制御不能な形で終了することは
  一切許されません。その場合、最終評価は 0 点です。該当するフラグを使ってください。
  設定ファイルが存在する場合を除き、ファイルを編集する必要は本来ありません。
  ファイルを編集したい場合は、その理由を評価される学生に明確に説明し、
  双方が納得していることを確認してください。

## 設定ファイル

### フォーマット

設定ファイルが要件に従っていることを確認してください。

- コメント行は '#' で始まる。
- 各行が 'KEY=VALUE' 形式である（小文字でも可）。
- 期待されるキーが存在する（WIDTH, HEIGHT, ENTRY, EXIT, OUTPUT_FILE, PERFECT）。

`はい / いいえ`

### エラー処理

設定ファイルを編集し、プログラムがエラーを正しく処理することを確認してください。

- 必須のキーを 1 つ削除する。
- 'KEY=VALUE' 形式に従わない行を追加する（例: '=' がない行）。
- 数値を文字に置き換える。
- 'PERFECT' に不正な真偽値を指定する。
- 'ENTRY' または 'EXIT' に不正な形式の整数タプルを指定する。

注意: プログラムが予期せず終了した場合、評価はそこで終了し、最終評価は 0 点です。

## 出力ファイル

### フォーマット

`a_maze_ing.py` を実行し、最初に生成（かつ表示）された迷路が、指定された出力
ファイルにも保存されていることを確認してください。以下が含まれているはずです。

- 'WIDTH' 文字からなる行が 'HEIGHT' 行、
- 空行 1 行、
- 'ENTRY' のタプル、
- 'EXIT' のタプル、
- 入口から出口までの最短経路を 'N', 'E', 'S', 'W' の方角で表した文字列。

課題に付属する検証スクリプトを使って、壁の整合性を確認してください。
出力ファイル中の最短経路の並びが、画面上の表示と一致することも確認してください。
1 つでも誤りがあれば、この設問は不合格となります。

`はい / いいえ`

## 迷路ジェネレータモジュール

### 迷路生成

迷路ジェネレータモジュールが課題の要求を満たしていること。

- ランダムに生成された迷路であること（適切な Python 標準モジュール、例えば
  `random` を使ってよい）。
- パラメータが矛盾している場合に適切に振る舞うこと（例: 入口や出口のセルが迷路の
  外にある、幅や高さが負である、など）。
- '42' パターンを構成するセルを除き、すべてのセルに到達できること。
- 迷路の四方が壁で囲まれていること。
- 壁のない 3x3 以上の開けた領域が存在しないこと。これをコード中でどう実装・
  検証したのかを質問してください。
- 迷路の中に '42' パターンがあること（サイズ的に不可能な場合を除く。その場合は
  ターミナルにメッセージが表示されること）。
- 設定ファイルの 'PERFECT' が True のとき、完全迷路（perfect maze）になること。
- 同じシードを使えば同じ迷路が再現できること（設定ファイルまたはコードを変更して
  確認する）。

これらの期待がすべて満たされていますか？

## ボーナス

### もっと欲しいですか？

ボーナスは、これまでのすべての設問に合格した場合にのみ考慮できます。
最大 5 個の異なる動作するボーナスを評価できます。各ボーナスは相応の作業量を
伴っている必要があります。

- 複数の迷路生成アルゴリズムをサポートする。
- 迷路生成中のアニメーションを追加する。
